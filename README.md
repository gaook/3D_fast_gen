# 单图生成 3D 模型 — 研究计划

> 目标：2026 年底前，使用 <7B 参数规模的生成模型，实现单张图片到高质量 3D 模型的生成，达到业界先进水平。

### 1.3 技术路线分类

```
单图 → 3D 生成技术路线
├── 路线 A：多视图生成 + 重建（两阶段）
│   ├── Stage 1: 单图 → 多视图一致图像（扩散模型，如 Zero123++, SV3D）
│   └── Stage 2: 多视图 → 3D 重建（前馈 LRM / NeuS / 3DGS）
│   └── 代表：InstantMesh, Hunyuan3D, Unique3D
│
├── 路线 B：端到端前馈重建
│   ├── 单图直接回归 3D 表示（NeRF triplane / 3DGS / Mesh）
│   └── 代表：LRM, TripoSR, LGM, GRM, Apple SHARP
│
├── 路线 C：3D 原生扩散/流匹配
│   ├── 在 3D 隐空间（体素/点云/结构化隐变量）上做生成
│   └── 代表：TRELLIS, TRELLIS.2, CLAY
│
└── 路线 D：混合方案
    ├── 3D 隐空间生成几何 + 2D 扩散生成纹理
    └── 代表：Hunyuan3D 2.0/2.5（DiT 生几何 + Paint 生纹理）
```

### 1.4 关键结论

- **<7B 完全可行**：TRELLIS.2（4B）和 TRELLIS 1.0（2B）已经是当前最强的开源模型之一
- **路线 C/D 是当前最优方向**：3D 原生隐空间生成在几何质量上明显优于两阶段方案
- **数据质量 > 数据数量**：TRELLIS 用 500K 精选数据训练，效果超越了用数百万数据的模型
- **3DGS 是最有前景的输出格式**：实时渲染、与你的 3dgs-viewer 完美衔接

### 阶段 1：数据工程（第 2–4 月，2026.04–06）

**目标：** 构建高质量训练数据集

| 任务 | 详情 | 预计时间 |
|------|------|----------|
| 数据收集 | 从 Objaverse-XL 筛选高质量子集（~500K–1M 物体） | 2 周 |
| 质量过滤 | 去除低质量、残缺、过小模型；基于渲染质量打分 | 2 周 |
| 多视图渲染 | Blender 渲染多视图 RGB + 法线 + 深度图（每物体 24–36 视角，512×512） | 3 周（GPU 集群） |
| 3D 隐空间处理 | 体素化 / 结构化隐变量编码（如果走路线 C） | 2 周 |
| 数据增强 | 随机光照、背景、相机参数扰动 | 1 周 |

**数据筛选策略（参考业界经验）：**
- 去除面数/顶点数过少的模型
- 去除非流形（non-manifold）和破损 mesh
- 使用 CLIP 打分移除抽象或无法识别的物体
- 使用 LAION aesthetic predictor 对渲染图打分过滤
- 典型结果：从 800K Objaverse 中筛出 ~200K–300K 高质量物体

**多视图渲染 pipeline（Blender）：**
- 相机分布：球面均匀采样，方位角 0–360°，仰角 -30° 到 +30°
- 每物体渲染 RGB + 深度 + 法线 + Alpha mask
- 光照：随机 HDRI 环境贴图（Poly Haven 等来源）
- 背景：白色或随机纯色

**数据集资源：**

| 数据集 | 规模 | 格式 | 用途 |

## 五、关键技术决策

### 5.1 3D 表示选择

| 表示 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **结构化隐变量（SLAT）** | 统一解码到多种格式、信息密度高 | 需要训练 VAE | ⭐⭐⭐⭐⭐ |
| 3D Gaussian Splatting | 实时渲染、与你的 viewer 兼容 | 非结构化，难以直接生成 | ⭐⭐⭐⭐ |
| NeRF Triplane | 紧凑、易于生成 | 渲染慢、分辨率受限 | ⭐⭐⭐ |
| 显式 Mesh | 工业标准、广泛兼容 | 拓扑不固定，难以直接回归 | ⭐⭐⭐ |

**推荐：** 采用 SLAT 作为生成目标，解码为 3DGS + Mesh 双输出。

### 5.2 生成范式选择

| 范式 | 代表 | 优点 | 缺点 |
|------|------|------|------|
| **Rectified Flow** | TRELLIS 系列 | 训练稳定、采样快 | 较新，社区经验少 |
| DDPM/扩散 | Hunyuan3D | 成熟、效果好 | 采样步数多 |
| 前馈回归 | LRM, SHARP | 极快推理 | 生成多样性受限 |

**推荐：** Rectified Flow Transformer，兼顾质量和速度。

### 5.3 图像编码器选择

| 编码器 | 参数量 | 特点 | 推荐度 |
|--------|--------|------|--------|
| **DINOv2-Large** | 0.3B | 强 3D 几何理解 | ⭐⭐⭐⭐⭐ |
| SigLIP | 0.4B | 强语义理解 | ⭐⭐⭐⭐ |
| DINOv2 + SigLIP 双编码器 | 0.7B | 几何 + 语义互补 | ⭐⭐⭐⭐⭐ |

## 六、关键技术挑战

| 挑战 | 描述 | 应对策略 |
|------|------|----------|
| **Janus 问题** | 多视图扩散模型生成不一致视角（如前后都有脸） | 3D 原生隐空间生成（路线 C）从根本上避免；或使用 epipolar attention |
| **纹理模糊** | 前馈模型往往产生模糊纹理 | 高分辨率训练（512→1024）、纹理细化阶段、PBR 材质预测 |
| **薄结构和细节** | 细长结构（如椅子腿、树枝）难以重建 | SDF 表示处理拓扑更好；增加几何分辨率 |
| **真实图片泛化** | 合成数据训练的模型在真实照片上泛化不足 | 使用 DINOv2/CLIP 编码器（更鲁棒）；引入 MVImgNet/CO3D 真实数据微调 |
| **背面幻觉** | 模型需要从单图推断看不见的部分 | 大规模数据训练建立先验；结合语义理解 |
| **Mesh 提取质量** | 从 NeRF/3DGS 提取干净 mesh 仍然困难 | FlexiCubes/DMTet 可微 mesh 提取；TRELLIS 的 SLAT 直接解码 mesh |
| **评估困难** | 没有单一指标能全面衡量 3D 生成质量 | 多指标组合 + 用户研究（human preference） |

## 九、评估指标体系

### 新视角合成质量
- **PSNR** — 像素级准确度（越高越好）
- **SSIM** — 结构相似性（越高越好）
- **LPIPS** — 感知质量（越低越好）

### 3D 几何质量
- **Chamfer Distance (CD)** — 预测与 GT 点云之间的双向最近点距离（越低越好）
- **F-Score** (@不同阈值，如 0.01) — 预测/GT 点在阈值内的比例（越高越好）
- **IoU** — 体积交并比（越高越好）

### 生成质量（无 GT 时）
- **FID** — 分布级别的生成质量
- **CLIP Score** — 图文/图图对齐度
- **用户研究** — 人类偏好评分（越来越重要）

### 常用评估基准
- GSO (Google Scanned Objects) 测试集 (~30 物体)
- OmniObject3D 测试集
- ABO (Amazon Berkeley Objects)
- Objaverse 留出测试集
